# data-prune-for-UATR
# Information-Aware Dynamic Sample Scheduling for Efficient Underwater Acoustic Target Recognition

This repository contains the official implementation of:

**Information-Aware Dynamic Sample Scheduling for Efficient Underwater Acoustic Target Recognition**

The paper proposes an information-aware dynamic sample scheduling method for underwater acoustic target recognition. Instead of using all densely segmented acoustic samples in every epoch, the proposed method dynamically selects a subset of training samples according to their estimated informativeness. This improves sample utilization, reduces redundant computation, and maintains or improves recognition accuracy.

## Overview

Training samples in underwater acoustic target recognition are usually generated by segmenting long-duration recordings into short audio clips. However, adjacent segments from the same recording often contain highly similar acoustic patterns, leading to redundant training samples. Conventional full-data training treats all samples equally in each epoch, which may result in inefficient sample utilization.

To address this issue, this work introduces an **Information-Aware Dynamic Sample Scheduling** strategy. The method maintains a lightweight loss memory for each training sample and dynamically reconstructs the training subset at each epoch. An exploration-to-focus schedule is used to balance data coverage in early training and informativeness-guided sample selection in later training.

## Key Features

* Dynamic sample scheduling for underwater acoustic target recognition
* Lightweight loss memory for sample informativeness estimation
* Exploration-to-focus training strategy
* Support for different sample retention ratios
* Support for waveform and Mel-spectrogram inputs
* Compatible with ResNet and ConvNeXt backbones
* Comparison with full-data training, random selection, OHEM, forgetting-based selection, and GradMatch
* Training efficiency analysis under different sample budgets

## Method

The proposed method consists of three main components:

### 1. Sample Informativeness Estimation

For each training sample, a loss memory value is maintained. The memory records the latest observed classification loss of the sample. A higher loss indicates that the current model has not sufficiently learned the corresponding acoustic pattern, so the sample may be more informative for training.

### 2. Exploration-to-Focus Scheduling

At each epoch, only a subset of samples is selected for optimization. The selected subset contains two parts:

* **Focused subset**: high-informativeness samples selected according to the loss memory.
* **Exploratory subset**: randomly selected samples used to preserve data coverage.

The exploration ratio gradually decreases during training. Early epochs emphasize broad data coverage, while later epochs focus more on informative samples.

### 3. Dynamic-Subset-Driven Optimization

The model is optimized only on the selected subset in each epoch. Unselected samples are not permanently discarded; they may be selected again in later epochs. This makes the method different from one-shot data pruning.

## Dataset

Experiments are conducted on the **DeepShip** underwater acoustic target recognition dataset.

The dataset contains ship-radiated noise recordings collected in real marine environments. Four target categories are used:

* Cargo
* Passenger ship
* Tanker
* Tug

To avoid data leakage, the dataset is split at the original audio-file level before segmentation.

## Data Preprocessing

All audio signals are processed as follows:

* Sampling rate: `16 kHz`
* Segment length: `3 s`
* Input representations:

  * Raw waveform
  * Mel spectrogram

For Mel-spectrogram extraction:

* Number of Mel filters: `128`
* FFT size: `1024`
* Hop length: `512`

## Experimental Settings

The following settings are used in the paper:

* Framework: PyTorch
* Python version: 3.8
* CUDA version: 11.8
* GPU: NVIDIA RTX 3090
* Optimizer: Adam
* Initial learning rate: `1e-3`
* Weight decay: `1e-5`
* Batch size: `32`
* Maximum epochs: `150`
* Label smoothing: `0.1`
* Learning rate scheduler: cosine annealing
* Sample retention ratios: `0.3`, `0.5`, `0.7`, `0.9`

## Data Preparation

Please organize the DeepShip dataset as follows:

```text
DeepShip/
├── Cargo/
├── Passenger/
├── Tanker/
└── Tug/
```

Before training, split the original audio files into training, validation, and test sets at the file level. Segmentation and feature extraction should be performed independently for each split to avoid data leakage.

Example preprocessing command:

```bash
python datasets/preprocessing.py \
    --data_root /path/to/DeepShip \
    --save_root /path/to/processed_deepship \
    --sample_rate 16000 \
    --segment_length 3
```

## Training

### Full-Data Training

```bash
python train.py \
    --config configs/resnet_wave.yaml \
    --mode full \
    --data_root /path/to/processed_deepship
```

### Random Selection

```bash
python train.py \
    --config configs/resnet_wave.yaml \
    --mode random \
    --retention_ratio 0.7 \
    --data_root /path/to/processed_deepship
```

### Proposed Dynamic Sample Scheduling

```bash
python train.py \
    --config configs/resnet_wave.yaml \
    --mode ours \
    --retention_ratio 0.7 \
    --epsilon_max 1.0 \
    --epsilon_min 0.3333 \
    --data_root /path/to/processed_deepship
```

## Evaluation

Evaluate a trained model on the test set:

```bash
python evaluate.py \
    --checkpoint /path/to/checkpoint.pth \
    --data_root /path/to/processed_deepship \
    --input_type wave
```

## Main Results

The proposed method improves recognition performance under reduced training budgets. Compared with random selection across all retention-ratio settings, it achieves an average absolute accuracy gain of **3.51 percentage points**.

Representative results on the DeepShip dataset include:

| Backbone | Input | Full Training | Random Selection |         Ours |
| -------- | ----: | ------------: | ---------------: | -----------: |
| ResNet   |  Wave |  63.26 ± 1.40 |     63.01 ± 2.09 | 67.50 ± 0.90 |
| ResNet   |   Mel |  68.59 ± 0.71 |     68.30 ± 1.27 | 70.78 ± 0.04 |
| ConvNeXt |  Wave |  62.46 ± 1.72 |     62.61 ± 1.11 | 66.56 ± 1.40 |
| ConvNeXt |   Mel |  65.06 ± 1.13 |     65.83 ± 0.96 | 67.11 ± 0.81 |

The method also improves training efficiency. When using only 30% of the training samples per epoch, it improves training-and-validation throughput by approximately **69%** while maintaining competitive recognition accuracy.

## Ablation Study

The paper studies the effects of different scheduling components:

* **Hard-only**: selects high-loss samples without explicit exploration.
* **Static Top-K**: fixes the selected subset after warm-up.
* **Fixed exploration ratio**: uses a constant exploration coefficient.
* **Ours**: combines loss memory, dynamic subset reconstruction, and exploration-to-focus scheduling.

The proposed full strategy achieves the best performance across different backbone-input settings.
